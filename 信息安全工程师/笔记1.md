# 2.网络攻击类型速记表

| 攻击类型                        | 核心定义                             | 攻击目的                    | 典型特征 / 关键词                      | 考试高频例子                                      |
| :-------------------------- | :------------------------------- | :---------------------- | :------------------------------ | :------------------------------------------ |
| **拒绝服务攻击（DoS/DDoS）**        | 通过消耗目标系统资源（带宽、CPU、内存），使其无法处理合法请求 | 让目标系统瘫痪、拒绝提供服务          | 系统变慢、响应超时、服务中断、大量无用请求           | SYN Flood、CC 攻击、DDoS 分布式攻击                  |
| **重放攻击（Replay Attack）**     | 截获并重复发送之前的合法数据包，冒充用户身份           | 绕过身份验证，冒充合法用户执行操作       | 重复发送历史请求、冒充身份、无需破解密码            | 截获登录请求后重复发送，非法登录系统                          |
| **中间人攻击（MITM）**             | 拦截通信双方的流量，篡改或窃取数据，双方却不知情         | 窃取敏感信息、篡改传输数据           | 通信被窃听、数据被篡改、会话劫持                | Wi-Fi 环境下窃取用户账号密码                           |
| **SQL 注入攻击**                | 向 Web 应用的输入框中插入恶意 SQL 语句，欺骗数据库执行 | 窃取、篡改或删除数据库数据，甚至控制服务器   | 输入框提交特殊字符（如 ' OR 1=1--）、数据库被拖库  | 登录框输入`admin' OR 1=1--`绕过登录验证                |
| **跨站脚本攻击（XSS）**             | 向网页中注入恶意脚本，在用户浏览器中执行             | 窃取用户 Cookie、冒充用户身份、挂马传播 | 网页被篡改、弹窗、用户 Cookie 被盗           | 论坛帖子中注入`<script>alert(1)</script>`窃取 Cookie |
| **跨站请求伪造（CSRF）**            | 诱导用户在已登录的状态下，点击恶意链接，执行未授权操作      | 冒充用户身份执行操作（如转账、改密码）     | 用户已登录状态、恶意链接、无需获取用户凭证           | 诱导用户点击链接，以用户身份修改账户信息                        |
| **缓冲区溢出攻击**                 | 向程序的缓冲区写入超出长度的数据，覆盖内存地址          | 执行恶意代码、获取系统权限           | 程序崩溃、权限提升、远程控制                  | 利用软件漏洞执行任意代码                                |
| **反射攻击（Reflection Attack）** | 利用第三方服务器放大攻击流量（DoS 的变种）          | 放大攻击效果，隐藏攻击者真实 IP       | 利用 NTP/DNS 服务器反射流量、DDoS 攻击的一种实现 | NTP 反射 DDoS 攻击                              |
| **暴力破解攻击**                  | 尝试所有可能的密码组合，破解用户账号密码             | 非法登录系统，获取权限             | 大量登录请求、字典攻击、弱口令破解               | 对 SSH/HTTP 登录接口进行密码穷举                       |
| **钓鱼攻击（Phishing）**          | 伪装成可信机构发送邮件 / 链接，诱导用户泄露敏感信息      | 窃取用户账号、密码、银行卡信息         | 伪装官方网站、虚假邮件、诱导点击链接              | 伪装成银行邮件，诱导用户输入账号密码                          |
#####  💡 易混攻击类型辨析（考试高频易错点）

1. **DoS vs DDoS**
	- DoS：单台设备发起攻击，威力有限
	- DDoS：多台设备（僵尸网络）同时发起攻击，威力更强，更难防御

1. **XSS vs CSRF**
	- XSS：注入脚本，直接窃取用户凭证（Cookie）
	- CSRF：利用用户已登录状态，诱导用户执行操作，不需要获取凭证

3. **重放攻击 vs 中间人攻击**
	- 重放攻击：重复发送历史请求，核心是 “重复”
	- 中间人攻击：拦截并篡改通信数据，核心是 “窃听 + 篡改”

---
##### 🛡️ 附：常见防御措施（额外加分点）

- **DoS/DDoS**：流量清洗、CDN 防护、限制单 IP 请求频率
- **SQL 注入**：参数化查询、输入验证、WAF（Web 应用防火墙）
- **XSS**：输入过滤、输出编码、设置 HttpOnly Cookie
- **CSRF**：使用 CSRF Token、验证 Referer、SameSite Cookie
- **重放攻击**：使用一次性会话 ID、时间戳验证、请求序号

#  3.什么是「平方剩余」？

简单来说，**平方剩余（Quadratic Residue）** 是模运算里的一个概念：
在模素数 p 的情况下，如果存在某个整数 y，使得$y^2 \equiv a \pmod{p}$ ，那么我们就称 a 是模 p 的**平方剩余**；反之，如果不存在这样的 y，则称 a 是模 p 的**非平方剩余**。

# 5.TCP/IP 四层模型

先记**从上到下四层顺序**：
**应用层 → 传输层 → 网际层 → 网络接口层**

## 一、先一句话区分：OSI 七层 vs TCP/IP 四层
OSI 是**理论教科书版**（分太细）
TCP/IP 是**现实互联网实际在用版**（合并精简）

## 二、用「寄快递」同一套逻辑，无缝对接 OSI，超好记

### 1. 应用层（最上层）

**对应 OSI：7 应用 + 6 表示 + 5 会话 三层合并**

人话：微信、浏览器、抖音、QQ、网盘

干什么：**你直接操作、产生数据的地方**

快递比喻：你写好信件、打包、密封、跟对方约好收发时间，全包了。

常用协议：HTTP、HTTPS、FTP、DNS、SMTP

### 2. 传输层

和 OSI**第 4 层一模一样**，没合并

人话：**TCP、UDP 干活**，管端口、管靠谱送货

- TCP：稳重可靠，怕丢包，要确认（微信聊天、网页）
- UDP：飞快不纠结，不管丢不丢（直播、游戏语音）
    快递比喻：快递公司，负责整车运输、保证件不丢、控制快慢。

### 3. 网际层（也叫互联网层）

对应 OSI**第 3 层**

人话：**IP 地址、路由器、选路找路**

只干两件事：给设备标 IP、帮数据包选走哪条路跨网段转发

快递比喻：填收件人详细地址、规划跨省运输路线。

常用协议：IP、ICMP、ARP

### 4. 网络接口层（最底层）

对应 OSI **2 数据链路 + 1 物理层 两层合并**

人话：网卡、交换机、网线、光纤、Wi-Fi

负责：把 IP 数据包变成**0 和 1 信号**，在局域网里按 MAC 地址转发、走物理线路

快递比喻：小区驿站分发 + 公路 / 铁路 / 运输车 跑路。

### 三、TCP/IP 四层 极简背诵表

| 层级  | 名字    | 对应 OSI | 核心作用           | 关键关键词           |
| --- | ----- | ------ | -------------- | --------------- |
| 4   | 应用层   | 7+6+5  | 用户应用、加密格式、会话连接 | HTTP / 微信 / APP |
| 3   | 传输层   | 4      | 端口、可靠传输        | TCP、UDP         |

| 1   | 网络接口层 | 2+1    | MAC 寻址、帧、网线信号  | 交换机、网卡、网线       |

# 6.IPv6
## 1. 什么是 IPv6？

**IPv6 = 互联网协议第 6 版**，用来替代 IPv4。

核心原因：**IPv4 地址不够用（32 位，约 43 亿）**，IPv6 彻底解决这个问题。

---

## 2. 最核心数据（必背）

- **IPv4：32 位**
- **IPv6：128 位**（2 的 128 次方个地址，几乎无限）
- **IPv6 基本报头：固定 40 字节**（IPv4 是 20~60 字节，更复杂）

---

## 3. IPv6 地址格式（必考）

### 格式

8 组，每组 4 个十六进制数，用 **冒号 `:`** 分隔

```
2001:0db8:0000:0000:0000:ff00:0042:8329
```

### 压缩规则（超常考）

1. **前导 0 可省略**
    `0db8` → `db8`
    
2. **连续全 0 的一段或多段，可用一个 `::` 代替**（**只能用一次**）
    上面例子可写成：

```
2001:db8::ff00:42:8329
```

---

## 4. IPv6 地址类型（高频）

IPv6 **没有广播地址**，只有三类：

1. **单播（Unicast）**：一对一通信
    - 全球单播：`2000::/3`（公网）
    - 链路本地：`fe80::/10`（内网，类似 169.254.x.x）
    - 唯一本地：`fc00::/7`（私有地址，类似 192.168.x.x）
    
2. **组播（Multicast）**：一对多，前缀 `ff00::/8`
3. **任播（Anycast）**：一对最近节点，同一地址分给多台设备

- `fe80::/10`：**fe80 开头** → 局域网本地用，出不了网段
- `fc00::/7`：**fc00 开头** → 企业家庭私网内网用
- `ff00::/8`：**ff 开头** → 群发组播地址
### 特殊地址（必背）

- `::1` → **环回地址**（类似 127.0.0.1）
- `::` → **未指定地址**（类似 0.0.0.0）

---

## 5. IPv6 的优势（选择题常考）

1. **地址无限**：解决 IPv4 枯竭
2. **安全内置**：**IPsec 强制支持**（IPv4 是可选）
3. **自动配置**：支持 **SLAAC**，插上网线自动生成 IP
4. **报头简化**：转发更快、路由器效率高
5. **无需 NAT**：地址足够，不用地址转换

---

## 6. 从 IPv4 到 IPv6 的过渡技术（常考）

1. **双栈（Dual Stack）**：同时跑 IPv4+IPv6（最常用）
2. **隧道（Tunnel）**：IPv6 数据包封装在 IPv4 里传输
3. **NAT64/DNS64**：让纯 IPv6 网络访问 IPv4 资源

---

## 7. 邻居发现协议 NDP（替代 IPv4 的 ARP）

- 用 **ICMPv6**
- 实现：地址解析、路由器发现、无状态自动配置

---

## 考试一句话总结（背这个就够用）

**IPv6 = 128 位、冒号十六进制、可压缩、无广播、有单播 / 组播 / 任播、内置 IPsec、自动配置、双栈过渡、NDP 替代 ARP。**
